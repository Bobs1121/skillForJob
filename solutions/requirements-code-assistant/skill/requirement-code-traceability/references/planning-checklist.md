# Implementation-plan checklist

Include:

1. Requirement scope and assumptions.
2. Existing behavior and evidence.
3. Affected modules and entry points.
4. Data and control flow.
5. Parameter and calibration changes.
6. State-machine changes.
7. Input/output and network-interface changes.
8. Fault, suppression, degradation, and fallback behavior.
9. Unit, integration, HIL, and vehicle-test cases as applicable.
10. Regression surface and unresolved questions.

Separate required work from optional refactoring.
